Client-Server-Prinzip

Der Client sendet eine Anfrage (Request) an einen Sever der Sever antwortet dann (Response). Der Client sendet die Request zum Beispiel in Form einer Webadresse (http-Request) der Server antwortet dann mit HTML oder CSS-Dateien, die vom Client verarbeitet werden und der dann die Website anzeigt.

IP-Adresse in zwei Versionen

Es gibt zwei Hauptversionen von IP-Adressen, die in Netzwerken verwendet werden: IPv4 (Internet Protocol Version 4): Dies ist die ältere Version von IP-Adressen und verwendet eine 32-Bit-Adresse, um Netzwerkgeräte zu identifizieren. Sie besteht aus vier durch Punkte getrennten Zahlen (z.B. 192.168.1.1). IPv6 (Internet Protocol Version 6): Dies ist die neuere Version von IP-Adressen, entwickelt, um den wachsenden Bedarf an Adressen zu bewältigen. IPv6 verwendet 128-Bit-Adressen, die in hexadezimaler Form dargestellt werden (z.B. 2001:0db8:85a3:0000:0000:8a2e:0370:7334).

Protokoll

In Netzwerken bezieht sich das Wort "Protokoll" auf eine Reihe von Regeln und Vereinbarungen, die festlegen, wie Daten zwischen Geräten in einem Netzwerk ausgetauscht werden. Protokolle definieren das Format der Daten, die Reihenfolge der Übertragung, Fehlerbehandlung und andere Aspekte der Kommunikation. Beispiele: HTTP (Hypertext Transfer Protocol) oder TCP (Transmission Control Protocol)

Router

Dient als Verbindungspunkt zwischen dem lokalen Netzwerk und dem Internet. Der Router sorgt dafür das die Datenpakete an die richtige IP-Adresse kommen und bietet parallel einen kabelgebundenen als auch drahtlosen Zugang. Dabei unterscheidet man zwischen vielen verschiedenen Arten, etwa einem Backbone-Router, Hardware-Router, Border-Router, High-End-Switches, Software-Router, Virtuelle Router oder einem DSL-Router.

Switch/Hub

Ein Switch wird zur direkten Weiterleitung eines Datenpacketes an einen spezifischen Empfänger genutzt, indem er die MAC-Adresse zuordnet und sich die passenden Wege merken kann. Dadurch ist er effizienter und kann ohne Probleme mehrere Geräte gleichzeitig kommunizieren lassen. Ein Hub hingegen sendet die empfangenen Daten immer an alle angeschlossenen Geräte, was allerdings sowohl wegen Sicherheitsbedenken als auch wegen der geringen Belastbarkeit des Gerätes wenig genutzt wird.

LAN und WLAN

LAN (Local Area Network) kann Daten nur mithilfe eines Kabels übertragen, WLAN (Wireless Local Area Network) überträgt die Daten kabellos per Funk. Dabei bietet LAN eine schnellere und sichere Verbindung, ist allerdings auf einen lokalen Bereich begrenzt. WLAN hingegen ermöglicht es den Geräten, ohne Kabel Internetzugang zu erlangen, ist allerdings auch anfälliger für Störungen und bietet oft eine eingeschränkte Geschwindigkeit im Gegensatz zu einem Kabelanschluss.

DHCP

DHCP (Dynamic Host Configuration Protocol) ist ein Netzwerkprotokoll, das automatisch IP-Adressen und andere Netzwerkkonfigurationen an Geräte in einem Netzwerk verteilt. Statt manuell IP-Adressen einzugeben, können Geräte im Netzwerk mithilfe von DHCP dynamisch temporäre IP-Adressen zugewiesen bekommen, wenn sie sich mit dem Netzwerk verbinden.

Gateway

Ein Gateway stellt die Schnittstelle zwischen unterschiedlichen Netzwerken dar und ermöglicht den Datenverkehr zwischen ihnen. Es fungiert als Übersetzer zwischen unterschiedlichen Protokollen, Netzwerktypen oder Topologien. Ein Gateway kann auf verschiedenen Ebenen arbeiten, z. B. auf der Anwendungsschicht, der Transport- oder der Netzwerkschicht.

Port

Ein Port ist eine numerische Kennung, die verwendet wird, um bestimmte Dienste oder Anwendungen in einem Computernetzwerk zu identifizieren. Ports ermöglichen es mehreren Anwendungen auf demselben Gerät, gleichzeitig Netzwerkverbindungen herzustellen, da der Datenverkehr anhand der Kombination aus IP-Adresse und Portnummer geroutet wird. Dabei sind einige Portnummern bereits für festgelegte Dienste reserviert, welche standardisiert etwa zum Beispiel für verschlüsselte und unverschlüsselte Internetkommunikation verwendet werden.

DNS und Domain

DNS (Domain Name System) ist ein System, das den Namen von Domänen (Webadressen) in numerische IP-Adressen umwandelt, die von Computern im Netzwerk verwendet werden. Eine Domain ist eine hierarchische Namensstruktur, die URLs (Uniform Resource Locators) in eine für Computer verständliche Form übersetzt. Dabei wird die Domain jeweils zum nächsten DNS-Server weitergeleitet, bis die Adresse in einem Verzeichnis auftaucht und übersetzt werden kann.

Routing und Routingtabellen

Routing bezieht sich auf den Prozess der Auswahl von Wegen für den Datenverkehr in einem Netzwerk. Dabei werden Routingtabellen verwendet, die Informationen darüber enthalten, wie Datenpakete von einem Netzwerksegment zum anderen weitergeleitet werden sollen. Diese Tabellen enthalten Einträge mit Zielnetzwerken, den nächsten Hops und möglicherweise Metriken, um den besten Weg zu bestimmen.

ISO/OSI-Modell

Das ISO/OSI-Modell steht für das Internationale Standardisierungsorganisation / Open Systems Interconnection Modell. Es ist eine theoretische Grundlage für die Entwicklung von Netzwerkprotokollen und -architekturen. Das Modell unterteilt die Kommunikation in sieben Schichten, von der physischen Übertragung bis zur Anwendungsebene, wobei jede Schicht spezifische Aufgaben und Verantwortlichkeiten hat.

Netztopologien

Netztopologien beschreiben die physische oder logische Anordnung von Netzwerkgeräten und -verbindungen. Es gibt verschiedene Topologien, darunter Bus, Stern, Ring, Mesh, Baum und Hybrid. Diese Topologien bestimmen, wie Geräte im Netzwerk miteinander verbunden sind und wie Daten fließen.
