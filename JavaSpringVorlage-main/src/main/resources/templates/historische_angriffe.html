NotPetya/Petya

Ablauf:

NotPetya trat in 2017 kurz nach den Attacken durch den Erpressungstrojaner WannaCry auf und machte schnell auf sich aufmerksam. Verbreitet über die ukrainische Steuersoftware MeDoc gehört es zu den ersten Trojanern, welche sich nicht durch Phishing oder Exploits, sondern innerhalb von Software verbreiten. Durch die hohe Verbreitung der Software wurden viele, auch multinationale Konzerne betroffen, einer davon Maersk. Der Angriff auf das System basierte dabei sowohl auf dem davor veröffentlichten EternalBlue-Exploit, welcher zwar von Microsoft gepatcht wurde, aber teils noch für Angriffe genutzt werden konnte, und einem Passwort-Dump, um sich auf den Rechnern einzuloggen.
Technische Hintergründe
Bei der Verbreitung geht NotPetya sehr präzise vor, indem ein Domainserver gesucht, infiziert und anschließend die spezifischen Computer angegriffen werden. Einmal auf dem System werden verschiedene Wege ausprobiert, darunter etwa der Eternalblue-Exploit, eine von der NSA entwickelten Hintertür, die zusammen mit mehreren anderen Zero-Day-Exploits zuvor durch die Hackergruppe Shadow Brokers veröffentlicht wurden. Zum Zeitpunkt des Angriffs hatte Microsoft bereits einen Patch veröffentlicht, allerdings waren Computer ohne das Update weiterhin anfällig, wobei auch Server etwa bei Maersk noch auf stark veralteten Systemen liefen. Weiterer Wege sind die Ausführung über Adminrechte und PsExec oder auch das Starten über die Windows Management Instrumentation Command-line (WMIC), wobei der Zugriff durch ein Passwort-Dump, welches etwa Mimikatz ähnelt, erlangt werden kann. Einmal ausgeführt wird das System neu gestartet und mit AES-128 oder RSA verschlüsselt, wobei nur eine relativ kleine Anzahl an Dateien verschlüsselt wird. Auffällig ist, dass eine Art Kill-Switch existiert, da NotPetya nicht ausgeführt wird, wenn eine bestimmte Datei auf dem System vorhanden ist. Dies bietet einen grundsätzlichen Schutz, ansonsten sind Updates, Backups und isoliertere Systeme den besten Schutz.

WannaCry

Ablauf:

WannaCry ähnelt in vielerlei Hinsicht NotPetya, wobei es auch als Vorgänger bezeichnet werden könnte. WannaCry verbreitet sich dabei auch als Wurm, wird allerdings klassischer per Mail übertragen und das System großflächiger verschlüsselte als sein Nachfolger. Auch die betroffenen Gruppen unterschieden sich folglich, da diesmal nicht mehr Konzerne im Mittelpunkt standen, sondern in diesem Fall etwa das NHS zu den größten betroffenen Netzwerken zählte.
Technische Hintergründe
WannaCry basiert wie NotPetya auch auf dem EternalBlue-Exploit, kann dadurch aber auch durch Updates gestoppt werden. Der Wurm selbst verbreitet sich über das Filesharing-Programm SMBv1 und installiert sich anschließend durch ein Backdoor-Tool namens “DoublePulsar“. Einer der größten Unterschiede liegt bei dem Kill-Switch, wobei das Programm die Verbindung zu einer spezifischen Webadresse prüft. Nachdem dies herausgefunden werden konnte, wurde die Adresse aktiviert, wobei die Verbreitung nicht gestoppt wurde, das Programm sich aber nicht mehr auf den betroffenen Rechner ausgeführt wird.
